RSA简介

RSA（Rivest、Shamir、Adleman）加密算法使用两个密钥：Private 和 Public。

方案 A

假设 Alice 想向 Bob 发送一条消息（仅供他查看！）。她可以使用 RSA 算法和 Bob 的公钥加密消息，这不是秘密（这就是为什么他们称它为 Public……）。一旦消息被加密，除了持有匹配私钥的人（即 Bob）之外，没有人可以解密它。

方案 B

反之亦然：如果 Alice 使用自己的Private Key加密消息，Bob（和 Eve，以及可以访问此“加密”消息的每个人）都可以使用 Alice 的公钥对其进行解密。那么，如果每个人都可以解密它，那么首先用私钥加密消息有什么意义呢？好吧，如果 Bob 想要确保消息是由 Alice 而不是其他人（夏娃？）写的，那是有道理的。

.NET RSACryptoServiceProvider

.NET Framework 在RSACryptoServiceProvider类中实现了 RSA 算法。此类的实例允许您创建密钥对、 使用公钥加密、使用私钥解密（如在第一个场景中）、签名（类似于第二个场景，但不完全是），并验证签名。

Sign 方法接受消息（作为字节数组）并为此特定数据创建签名。在第二种情况下，Alice 可以给 Bob 写一条消息，并使用这种方法用她自己的私钥获取签名。然后，她可以将带有签名的消息按原样（未加密）发送给 Bob。为了验证作者 ID（Alice），Bob 将使用VerifyAlice 的公钥为： 的方法，如果这是 Alice 编写和签名的原始消息Verify(aliceMessage, aliceSignature)，他将得到“ ”，或者如果从那时起甚至更改了一位，他将得到“ ” . 这是私钥加密的一种有用实现，但有时它太复杂了。您可能只想发送一条小消息，以便接收者可以解密它并确保它来自您，truefalse

使用 RSAEncryption 类

该类有六个主要方法：

复制代码
void LoadPublicFromXml(string publicPath)
void LoadPrivateFromXml(string privatePath)
byte[] PrivateEncryption(byte[] data)
byte[] PublicEncryption(byte[] data)
byte[] PrivateDecryption(byte[] encryptedData)
byte[] PublicDecryption(byte[] encryptedData)

我相信方法名称是不言自明的。RSACryptoServiceProvider首先，您必须使用 .NET类创建一个私钥/公钥对。为此，您只需创建此类的一个实例，然后调用适当的方法，如下所示：

C＃
复制代码
void LoadPublicFromXml(string publicPath)
RSACryptoServiceProvider rsa = new RSACryptoServiceProvider();
File.WriteAllText(@"C:\privateKey.xml", rsa.ToXmlString(true));  // Private Key
File.WriteAllText(@"C:\publicKey.xml", rsa.ToXmlString(false));  // Public Key

// Then, you can load those files to RSAEncryption instance:
RSAEncryption myRsa = new RSAEncryption();
myRsa.LoadPrivateFromXml(@"C:\privateKey.xml");
myRsa.LoadPublicFromXml(@"C:\publicKey.xml");

// Once the keys are loaded (if you load a private key, there is no need to
// load the public one) you can start Encrypt / Decrypt data
// using Private / Public keys.
byte[] message = Encoding.UTF8.GetBytes("My secret message");
byte[] encryptMsg = myRsa.PrivateEncryption(message);

byte[] decryptMsg = myRsa.PublicDecryption(encryptMsg);
string originalMsg = Encoding.UTF8.GetString(decryptMsg);
// returns "My secret message"

更新：新版本

RSA 私有加密的新实现有几个优点：

  1. 错误修复：添加了随机填充以支持 0 字节前缀数据。
  2. 使用新的 .NET 4 " BigInteger" 结构来支持数学。
  3. 扩展方法实现：唯一需要的类实例是RSACryptoServiceProvider.
  4. 更好的异常和错误处理。
  5. 添加了 UnitTest 项目。
  6. 一般来说，更优雅的代码（我希望..！）。
