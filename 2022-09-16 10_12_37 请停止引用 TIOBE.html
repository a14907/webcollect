什么是 TIOBE？

TIOBE 编程社区指数是编程语言流行度的一个指标。该索引每月更新一次……重要的是要注意，TIOBE 索引并不是关于 最好的 编程语言或 编写大多数代码行的语言 。

该索引可用于检查您的编程技能是否仍然是最新的，或者在开始构建新软件系统时就应该采用哪种编程语言做出战略决策。

资料来源：tiobe.com

帮助我们选择应该使用哪种语言的编程语言索引。听起来很有用！

那么TIOBE是如何计算这个指数的呢？您可能不相信这一点，但他们会计算每种编程语言的搜索引擎结果数量。不是查询的人数，不是他们进行的查询的数量，也不是情绪。它完全依赖于搜索引擎报告的那个无用的数字。（来源：TIOBE 指数定义）

为了给自己披上数据驱动的外衣，他们不只检查一个搜索引擎 (google.com)，而是检查多个搜索引擎 (bing.com、google.de、google.co.in 和其他一些)。但是 25 个无用的来源并不比 1 个无用的来源好。

尽管它有2490 万个Google 结果，但我不推荐使用这种语言进行编程。

TIOBE 的荒谬之处

那么我们看到这个垃圾输入会产生什么样的输出呢？

让我们看看广泛的趋势。也许这将使我们能够做出战略决策。

在 2016 年和 2017 年，索引 Java（绿色）和 C（黑色）中排名前 2 位的语言的趋势线崩溃了。Java 下降了 42%。C 甚至更糟 - 62%。发生了什么，是需求崩溃还是供应崩溃？没有其他语言会以他们为代价获得收益，所以也许我们看到了软件行业的衰退？我们是否应该优先考虑重写我们的 Java 应用程序？

事实证明，C 和 Java 在 2018 年都出现了大幅增长，恢复了 2016 年的水平。我猜谷歌正在修补他们的搜索算法。我希望没有人根据这些数据做出任何决定。

也许我们可以看看特定语言的排名，而不是趋势。

  • 在任何可通过的指标上，例如 Github PR、StackOverflow 问题、开发人员调查或职位发布，JavaScript 都是排名第一的语言。根据 TIOBE，JavaScript (#7) 小于 Visual Basic (#6)。
  • 现在您可能想知道，为什么 Visual Basic 这么高？以前不是这样的。2020 年 3 月 4 日 - 0.77%。但是当世界进入封锁状态时，VB 的受欢迎程度呈爆炸式增长。29 天后，它增长了 6 倍，达到 4.72%。从那以后它一直处于上升趋势。（顺便说一句，没有其他来源能够支持这一点。不是谷歌搜索趋势，或者StackOverflow 问题或其他任何东西。只有 TIOBE 认为这发生了。）
  • 将这种荒谬放在上下文中，Visual Basic 是 Swift (1.27%) 和 Objective-C (0.94%)总和的两倍多。整个 iOS、iPadOS、watchOS、macOS 生态系统的规模显然只有强大的 Visual Basic 生态系统的一半。
  • 汇编语言（#8）要么太高，因为没有人在汇编中编程，要么太低，因为不是所有的编程最终都是汇编？无论哪种方式，排名都是错误的。
  • 可悲的是，Logo (#48) 编程市场似乎已经走下坡路。在它的全盛时期，它在 TIOBE 上的排名高达第 21 位。这是涉及在屏幕上移动海龟的编程语言。

我们可以继续下去。我们可以问为什么 TypeScript 在 #37 上表现不佳，远远落后于 Delphi/Object Pascal (#14) 和 Scratch (#21)。但是你明白了，这一切都没有意义，除非是偶然的。

是否有可能即使输出完全错误，他们仍然幸运地做出正确的预测？再次，不。在2012 年 3 月的 TIOBE 报告中，他们对 2022 年流行的语言 Go 持悲观态度。他们还将 Python 排在低位，无论以任何标准衡量，它都是当今排名前三的语言。

奇怪的是，你在他们的网站上找不到带有这些坏预测的旧报告。

评估项目的语言

幸运的是，有更好的方法来评估语言。

对于项目，请查看技术匹配度。

  • 可以雇用/培训开发人员使用它吗？
  • 他们会乐意使用它吗？
  • 我们在这种语言上的开发速度是多少？保持它在生产中运行的运营负担是什么？
  • 我们需要依赖第三方代码吗？该代码是否已经存在，它的质量如何？

只有您的团队才能回答这些问题。这取决于您的业务问题、您当地的就业市场、您的工程预算。如果你雇佣了聪明人，他们可以解决这些问题。

在较小程度上，您的团队需要考虑长期趋势。该语言在未来也必须保持良好的技术适应性。如果您今天可以找到开发人员，但最终不得不乞求程序员在几年内退休，那就不好了。

预测未来很难。但是你可以看看

  • 开发商调查。
    • StackOverflow 年度调查- 最常用、最喜爱和最想要的语言。
    • JetBrains - 最流行、发展最快的语言。
  • Github 趋势。
    • Octoverse 状态- 顶级语言。
    • Github PRs（按语言）
  • StackOverflow 人们所询问的趋势。
    • 比较Java、C++、C、Python
    • 比较Kotlin、Rust、TypeScript、Swift
  • 谷歌搜索人们搜索的趋势。
    • 比较Java、C++、C、Python
    • 比较Kotlin、Rust、TypeScript、Swift

作为开发人员评估语言

与需要选择一种语言并坚持使用的项目不同，您不需要这样做。如果您将您的身份附加到一种语言，请放弃它。您是开发人员，而不是 $language 开发人员。

随着时间的推移扩大您的工具箱。你最喜欢的编程语言并不能解决你将来遇到的每一个问题，所以要定期学习新的。如果您的工具箱只包含一把锤子，那么一切都开始看起来像钉子。

如果您使用过垃圾收集语言，请尝试使用手动内存管理的语言，反之亦然。如果您主要使用命令式语言，请尝试使用函数式结构的语言，反之亦然。分析您知道的语言并尝试提供不同的语言。

但无论您做什么，都不要根据 Google 结果的数量做出决定。

Reddit 上的评论指出了之前批评 TIOBE 的文章。它还解释了为什么像 Delphi 这样的未知语言在该索引中如此高（#14）。这是因为Delphi 社区在操纵 TIOBE 排名方面付出了很多努力。对他们有好处。
